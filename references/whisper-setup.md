# Whisper Setup And ASR Usage

Use this reference when the ASR stage fails because Whisper, PyTorch, ffmpeg, CUDA, or model weights are missing. Official references: [OpenAI Whisper](https://github.com/openai/whisper) and [PyTorch local install selector](https://pytorch.org/get-started/locally/).

## What The ASR Script Expects

`asr_batch.py` transcribes videos under `video/` and writes:

```text
asr/
  transcript/
    {视频名}.json
    {视频名}.srt
    {视频名}.txt
  asr_status.jsonl
  asr_errors.log
```

Downstream scripts read `asr/transcript/{视频名}.srt`, so keep `--output asr` unless you also override later stages.

## Install Runtime

Install Python packages in the environment that will run `asr_batch.py`:

```powershell
python -m pip install -U pip
python -m pip install -U openai-whisper openpyxl
```

Install PyTorch separately with the command generated by the PyTorch install selector for the target machine. CPU-only is simplest; CUDA is much faster on a compatible NVIDIA GPU.

CPU example:

```powershell
python -m pip install torch torchvision torchaudio
```

CUDA example shape, verify the exact CUDA version on the PyTorch site before using:

```powershell
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

Install `ffmpeg` and ensure both `ffmpeg` and `ffprobe` are on `PATH`:

```powershell
ffmpeg -version
ffprobe -version
```

On Windows, common options are `winget install Gyan.FFmpeg`, Chocolatey, Scoop, or a manually downloaded ffmpeg build added to `PATH`.

## Check Installation

Run:

```powershell
python -c "import torch, whisper; print('torch', torch.__version__, 'cuda', torch.cuda.is_available()); print(whisper.available_models())"
```

Expected result:

- `torch` imports without error.
- `whisper.available_models()` prints model names such as `tiny`, `base`, `small`, `medium`, `large-v3`, `large-v3-turbo`, and `turbo`.
- `cuda True` means GPU decoding is available; `cuda False` means use CPU or reinstall a CUDA-enabled PyTorch build.

## Download Weights

Whisper downloads official model weights automatically the first time `whisper.load_model()` runs. To pre-download into a known cache directory:

```powershell
$cache = "$env:USERPROFILE\.cache\whisper"
python -c "import whisper; whisper.load_model('large-v3-turbo', download_root=r'$env:USERPROFILE\.cache\whisper')"
```

For a smaller/faster test model:

```powershell
python -c "import whisper; whisper.load_model('base', download_root=r'$env:USERPROFILE\.cache\whisper')"
```

Typical cached filenames include:

```text
C:\Users\<you>\.cache\whisper\base.pt
C:\Users\<you>\.cache\whisper\large-v3-turbo.pt
```

Use `large-v3-turbo` or `turbo` for a good speed/quality balance. Use `base` or `small` for quick pipeline tests.

## Run ASR

Recommended full run:

```powershell
python asr_batch.py --input video --output asr --model-name large-v3-turbo --model-path "$env:USERPROFILE\.cache\whisper\large-v3-turbo.pt"
```

The script can load by official `--model-name` and will use `--model-path`'s parent directory as the cache/download directory. If using a custom `.pt` checkpoint that is not an official Whisper model name, omit `--model-name` and point `--model-path` directly at the file:

```powershell
python asr_batch.py --input video --output asr --model-path "D:\models\whisper-custom.pt"
```

Plan before a long run:

```powershell
python asr_batch.py --input video --output asr --model-name base --model-path "$env:USERPROFILE\.cache\whisper\base.pt" --dry-run
python asr_batch.py --input video --output asr --model-name base --model-path "$env:USERPROFILE\.cache\whisper\base.pt" --limit 2
```

Force regeneration:

```powershell
python asr_batch.py --input video --output asr --model-name large-v3-turbo --model-path "$env:USERPROFILE\.cache\whisper\large-v3-turbo.pt" --overwrite
```

## Device And Precision

Default device is `auto`, which uses CUDA when PyTorch reports a GPU and CPU otherwise.

Useful overrides:

```powershell
--device cuda
--device cpu
--fp16 false
```

Use `--fp16 false` on CPU or when CUDA fp16 causes instability. The script's default `--fp16 auto` enables fp16 only on CUDA.

## Troubleshooting

- `ModuleNotFoundError: whisper`: install `openai-whisper` in the active Python environment.
- `No such file or directory: ffmpeg`: install ffmpeg and make sure the shell can run `ffmpeg -version`.
- `torch.cuda.is_available()` is `False`: install a CUDA-enabled PyTorch build that matches the machine's NVIDIA driver/CUDA support, or run with `--device cpu`.
- `Model checkpoint not found`: pass `--model-name` with an official model name so the script can download/cache it, or pass a real `--model-path`.
- Very slow ASR: use a smaller model for testing, enable CUDA, or run with `--limit` while checking the rest of the pipeline.
